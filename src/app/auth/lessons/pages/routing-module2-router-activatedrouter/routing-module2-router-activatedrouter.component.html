Interview Questions & Answers – Module 2

Q1. What is the difference between Router and ActivatedRoute?

Answer: Router is used to navigate to a different route. ActivatedRoute is used to read information about the current active route such as parameters, query parameters, fragments, and static route data.

Q2. How do you access query parameters?

Answer: Query parameters can be accessed in four ways:

  • route.queryParamMap — reactive and type safe
  • route.queryParams — reactive, plain object
  • route.snapshot.queryParamMap — one-time read
  • route.snapshot.queryParams — one-time read, plain object

Q3. How do you access static route data?

Answer: Static route data can be accessed using:

  • route.data — reactive, preferred
  • route.snapshot.data — one-time read
  • route.parent.data — reading data from parent route
  • route.firstChild.data — reading data from child route

Q4. When should you use route parameters?

Answer: Route parameters are used when the value identifies a resource, such as user ID, product ID, or item ID.

Q5. When should you use query parameters?

Answer: Query parameters are used for filters, search, sorting, pagination, and other optional values that do not identify a resource.

Q6. What is a fragment and when do you use it?

Answer: A fragment is the part after the hash in the URL. It is used to jump or scroll to a specific section on a page, such as a top anchor or a highlighted block.

Q7. What is navigation state?

Answer: Navigation state allows passing temporary data during routing without exposing it in the URL. It is suitable for items like a cart, form data, or selected items. It does not survive page refresh.

Q8. What is the difference between snapshot and observable when reading parameters?

Answer: Snapshot reads the values only once when the component loads. Observables update automatically when the route parameters change while the component is still active.

Q9. What is relative navigation?

Answer: Relative navigation allows navigating relative to the current route instead of starting from the root. It is used inside nested routes and child modules.

Q10. What is the difference between queryParamMap and queryParams?

Answer: queryParamMap is type safe and provides helper methods like get. queryParams returns a plain object. queryParamMap is recommended for cleaner, safer code.
