Module 1 – Angular Routing Basics

Internal Working of Angular Routing

Angular Routing internally works as a state machine that listens to URL changes and loads components dynamically without refreshing the page. Here is the flow of how routing actually works behind the scenes:

  • 1. URL Parsing: The Router reads the browser URL and breaks it into segments.
  • 2. Route Tree Matching: It compares each segment with the route configuration from top to bottom.
  • 3. Navigation Events: Angular emits a sequence of navigation lifecycle events such as NavigationStart, RoutesRecognized, GuardsCheckStart, ResolveStart, and NavigationEnd.
  • 4. Guard Execution: If the route has guards, Angular executes authentication or permission checks.
  • 5. Resolver Execution: If resolvers exist, Angular fetches required data before loading the component.
  • 6. Dynamic Component Creation: The matched component is created dynamically using Angular’s dynamic component loader and inserted inside <router-outlet>.
  • 7. History API: Angular uses the HTML5 History API to update the browser URL without reloading the entire page.
  • 8. View Rendering: The final component is rendered inside the router-outlet, completing the navigation.

This entire process happens without any full page reload, which makes Angular applications behave like fast, smooth single-page applications.

1. What is Angular Routing?

Angular Routing allows navigation between different views in a Single Page Application without reloading the entire page. It maps URLs to components and manages UI state.

2. How does Angular match routes?

Angular matches routes from top to bottom and selects the first matching route. Specific routes must be placed before generic or dynamic ones.

3. What is <router-outlet>?

<router-outlet> is a directive provided by Angular Router that acts as a placeholder where the router dynamically loads the component based on the current route configuration.

41. Absolute vs Relative Navigation

Absolute Navigation

Previous URL: /admin/users/12/details

Navigation Code: 

this.router.navigate(['/products']);

Updated URL: /products (Always starts from the root)

Relative Navigation

Previous URL: /products/10/specs

Navigation Code: 

this.router.navigate(['../reviews'], {{ '{' }} relativeTo: this.route {{ '}' }});

Updated URL: /products/10/reviews (Relative to the current route)

5. Wildcard Route

Used to handle unknown URLs (404 page).

{{ '{' }} path: '**', component: PageNotFoundComponent {{ '}' }}

Intermediate Concepts

6. How Child Routes Work

Child routes allow nested routing. Parent component must contain its own router-outlet.


{{ '{' }}
  path: 'admin',
  component: AdminLayoutComponent,
  children: [
    {{ '{' }} path: 'users', component: UserListComponent {{ '}' }}
  ]
{{ '}' }}
7. Lazy Loading

Lazy loading loads feature modules only when the route is visited.


{{ '{' }}
  path: 'admin',
  loadChildren: () =>
    import('./admin/admin.routes').then(m => m.ADMIN_ROUTES)
{{ '}' }}
8. Why pathMatch: 'full'?

Used to match the full empty path during redirects. 

{{ '{' }} path: '', redirectTo: 'home', pathMatch: 'full' {{ '}' }}

Senior Level Concepts

9. Navigation Lifecycle Events

Angular triggers a set of events during navigation:

  • NavigationStart
  • RoutesRecognized
  • GuardsCheckStart
  • ResolveStart
  • ResolveEnd
  • NavigationEnd
  • NavigationError
  • NavigationCancel
10. What happens when router.navigate() is called?
  • URL is parsed
  • Best route is matched
  • Guards run
  • Resolvers fetch data
  • Component is dynamically created
  • Inserted into router-outlet
  • History API updates URL without reload
11. Why route order matters?

Angular picks the first matching route. So specific routes must always be defined before dynamic ones.

12. How Angular prevents full page reloads?

Angular uses the HTML5 History API (pushState) to change URL without sending a request to the server. It handles routing internally and loads components dynamically.
